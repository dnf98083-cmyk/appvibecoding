# Claude Code 단계별 프롬프트 — 할 일 관리 앱

PRD를 5단계 프롬프트로 변환한 문서.
각 단계를 순서대로 Claude Code에 입력하면 앱이 완성된다.

---

## 단계 1 — 기본 구조 + 할 일 추가/삭제

```
20260323 폴더 안에 index.html 파일을 만들어 줘.

할 일 관리 앱의 기본 뼈대를 만들어야 해. 아래 조건을 지켜줘:

1. 순수 HTML + CSS + JavaScript만 사용 (외부 라이브러리 없음)
2. 상단에 앱 제목("✅ 할 일 관리")과 오늘 날짜 표시
3. 텍스트 입력창 + [추가] 버튼으로 할 일 등록
   - Enter 키로도 추가 가능
4. 각 항목에 삭제(✕) 버튼 배치
5. 모든 데이터는 LocalStorage에 저장 (새로고침해도 유지)

데이터 구조:
{ id, text, done, category, createdAt }

완료 기준: 할 일을 추가하고 새로고침해도 목록이 남아있어야 함.
```

---

## 단계 2 — 완료 체크 + 카테고리

```
index.html에 완료 체크와 카테고리 기능을 추가해 줘.

1. 각 항목 왼쪽에 체크박스 추가
   - 체크 시 텍스트에 취소선 표시
   - 완료된 항목은 목록 하단으로 자동 이동
2. 할 일 추가 시 카테고리 선택 드롭다운 추가
   - 업무 (파란색 뱃지)
   - 개인 (초록색 뱃지)
   - 공부 (보라색 뱃지)
3. 각 항목에 카테고리 뱃지 표시
4. LocalStorage 저장 구조에 category, done 필드 반영

완료 기준: 체크박스 클릭 시 취소선이 생기고 하단으로 이동해야 함.
```

---

## 단계 3 — 카테고리 필터 탭

```
index.html 상단에 카테고리 필터 탭을 추가해 줘.

1. [전체] [업무] [개인] [공부] 탭 버튼 4개
2. 탭 클릭 시 해당 카테고리 항목만 표시
3. 현재 선택된 탭은 강조 표시 (active 스타일)
4. 필터 상태는 페이지 이동 시 유지하지 않아도 됨
   (새로고침 시 [전체]로 초기화)

완료 기준: 탭 클릭 시 해당 카테고리 항목만 화면에 표시되어야 함.
```

---

## 단계 4 — 진행률 바 + 통계

```
index.html에 진행률 표시 기능을 추가해 줘.

1. 상단에 전체 진행률 바 표시
   - 형식: ████████░░░  12/18 완료 (67%)
   - 완료/추가 시 실시간 업데이트
2. 탭 옆에 카테고리별 완료 현황 숫자 표시
   - 예: [업무 3/5] [개인 2/4] [공부 1/3]
3. 완료된 항목 전체 삭제 버튼 추가 (하단)
   - 완료 항목이 없을 땐 버튼 비활성화

완료 기준: 체크박스 토글 시 진행률 바 수치가 즉시 바뀌어야 함.
```

---

## 단계 5 — 인라인 수정 + 마감일 + UI 완성

```
index.html에 마지막 기능을 추가하고 UI를 완성해 줘.

1. 인라인 수정
   - 할 일 텍스트 더블클릭 시 편집 가능
   - Enter 또는 바깥 클릭 시 저장
   - Escape 시 취소

2. 마감일 설정
   - 할 일 추가 시 날짜 입력 필드 (선택 사항)
   - 마감일이 있으면 항목에 D-day 표시
     - 오늘: "D-day" (빨간색)
     - 지남: "D+N" (회색)
     - 남음: "D-N" (파란색)

3. UI 마무리
   - 전체 배경, 카드, 버튼 스타일 통일
   - 할 일이 없을 때 빈 상태 메시지 표시
     ("+ 버튼으로 첫 번째 할 일을 추가해보세요")
   - 모든 기능이 LocalStorage와 연동되는지 최종 확인

완료 기준:
- 텍스트 더블클릭 시 편집 모드 진입
- 마감일 D-day가 색상과 함께 표시
- 새로고침 후 모든 데이터 유지
```

---

## 사용 방법

1. Claude Code 실행
2. 단계 1 프롬프트 입력 → 실행 확인
3. 단계 2 프롬프트 입력 → 실행 확인
4. 순서대로 단계 5까지 반복
5. 각 단계 완료 후 브라우저에서 index.html 열어서 테스트
