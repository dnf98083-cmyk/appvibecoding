# /add-question — 새 퀴즈 문항 추가

새로운 퀴즈 문항을 생성하고 할루시네이션 검증 후 questions.json에 추가합니다.

## 사용법
```
/add-question [카테고리] [난이도]
```
예: `/add-question 과학 어려움`

카테고리와 난이도를 지정하지 않으면 기존 문항 분포를 보고 부족한 영역을 채웁니다.

## 실행 단계

1. **현재 문제 은행 분석**
   - questions.json을 읽어 카테고리별/난이도별 분포 확인
   - 부족한 영역 파악

2. **문항 생성 (할루시네이션 방지 원칙 적용)**
   다음 형식으로 문항을 생성합니다:
   ```json
   {
     "id": [다음 번호],
     "category": "[카테고리]",
     "difficulty": "[난이도]",
     "question": "[문제]",
     "choices": ["정답", "오답1", "오답2", "오답3", "오답4"],
     "answer": 0,
     "explanation": "[상세 해설 — 왜 정답인지 논리적 설명]",
     "source": "[구체적 출처 1], [구체적 출처 2]",
     "confidence": [0.0~1.0]
   }
   ```

3. **자체 검증**
   생성 전 스스로 체크:
   - [ ] 정답이 확실히 맞는가? (수치/이름 재확인)
   - [ ] 오답 선택지가 그럴듯하지만 틀린가?
   - [ ] 출처가 2개 이상이고 구체적인가?
   - [ ] 확실하지 않으면 confidence를 0.7 미만으로 설정했는가?

4. **questions.json에 추가**
   검증된 문항을 questions 배열 끝에 추가합니다.

5. **validator.py 재실행으로 최종 확인**
   ```
   python -X utf8 validator.py
   ```

## 주의사항
- confidence가 0.7 미만이면 반드시 [검증 필요] 사유를 명시합니다
- AI가 생성한 통계/수치는 항상 confidence 0.8 이하로 설정합니다
- 정치적으로 민감한 주제는 추가하지 않습니다
