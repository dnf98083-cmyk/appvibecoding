## 교차 검증 가이드라인 (할루시네이션 방지)

Claude Code가 퀴즈 문항을 생성하거나 수정할 때 반드시 아래 규칙을 따른다.

### 신뢰도(confidence) 기준
- **0.9 이상** : 공신력 있는 문헌/공식 데이터로 확인된 사실
- **0.7~0.9** : 일반적으로 통용되나 세부 이견이 있을 수 있음
- **0.7 미만** : AI가 생성했거나 미검증 정보 → 반드시 교차 검증 필요

### 출처(source) 작성 원칙
- "Wikipedia" 단독 사용 금지
- 구체적인 문헌명 또는 기관명 명시
- 가능하면 2개 이상의 독립 출처 기재

### 문항 생성 시 자체 체크리스트
- [ ] 수치(날짜, 거리, 점수 등)를 공식 자료로 재확인했는가?
- [ ] 고유명사(인명, 지명, 작품명)의 표기가 정확한가?
- [ ] 정답이 선택지 중 단 하나만 명확히 맞는가?
- [ ] 해설이 정답을 논리적으로 뒷받침하는가?
- [ ] confidence가 0.7 미만이면 explanation에 "※ 검증 필요" 표기했는가?

### 교차 검증 절차
1. 문항 생성 후 `python -X utf8 validator.py` 실행
2. 경고(⚠️) 문항은 1차 자료로 수동 확인
3. 확인 완료 후 confidence 점수 업데이트
