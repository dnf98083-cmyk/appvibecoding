# 상식 퀴즈 게임 — Claude Code 구현 프롬프트 3단계

PRD.md를 바탕으로 Claude Code에게 순서대로 입력하는 프롬프트입니다.
각 단계를 완료한 후 다음 단계로 넘어가세요.

---

## 1단계 프롬프트 — 문제 은행 생성

```
PRD.md를 읽고 questions.json 파일을 만들어줘.

조건:
- 카테고리: 한국사, 과학, 지리, 일반상식
- 각 카테고리마다 10문제씩, 총 40문제
- 각 문제는 아래 형식을 따를 것

{
  "id": 1,
  "category": "한국사",
  "question": "문제 내용",
  "choices": ["보기1", "보기2", "보기3", "보기4"],
  "answer": 0,
  "explanation": "정답 해설"
}

- choices는 반드시 4개
- answer는 정답 보기의 0-based 인덱스 (0~3)
- explanation은 왜 정답인지 한 문장 이상 설명
- 문제는 사실에 기반하고 출제 의도가 명확할 것
```

---

## 2단계 프롬프트 — 게임 로직 구현

```
PRD.md와 questions.json을 읽고 quiz_game.py를 만들어줘.

구현할 기능:
1. 시작 메뉴
   - 닉네임 입력
   - 모드 선택: (1) 전체 랜덤  (2) 카테고리 선택

2. 게임 진행
   - 선택한 카테고리(또는 전체)에서 문제 랜덤 셔플 후 출제
   - 4지선다 보기 출력, 번호(1~4) 입력 받기
   - 입력 즉시 정답/오답 피드백
     - 정답: "정답입니다!" + 격려 메시지
     - 오답: "오답입니다. 정답은 [정답]" + 해설 출력

3. 게임 종료
   - 최종 점수 계산 (정답 1개 = 10점)
   - 등급 표시: S(380~400) / A(300~370) / B(200~290) / C(100~190) / D(0~90)
   - rankings.json에 {닉네임, 점수, 등급, 날짜} 저장

4. 랭킹 조회
   - rankings.json에서 상위 10명 불러와 출력

- Python 표준 라이브러리만 사용 (외부 패키지 없이)
- Windows 터미널 한글 깨짐 방지: sys.stdout utf-8 설정 포함
```

---

## 3단계 프롬프트 — 테스트 작성 및 검증

```
quiz_game.py와 questions.json을 읽고 tests/test_quiz.py를 pytest로 작성해줘.

테스트 항목:
1. 문제 은행 검증
   - questions.json에 40문제가 있는가
   - 카테고리별로 정확히 10문제씩인가
   - 모든 문제에 choices 4개, answer(0~3 범위), explanation이 있는가

2. 게임 로직 검증
   - 점수 계산이 올바른가 (정답 수 × 10)
   - 등급 판정이 PRD 기준과 일치하는가
     - 400점 → S, 350점 → A, 250점 → B, 150점 → C, 50점 → D

3. 랭킹 기능 검증
   - 결과가 rankings.json에 저장되는가
   - 상위 10명만 반환되는가
   - 점수 내림차순으로 정렬되는가

작성 후 pytest를 실행하고 결과를 알려줘.
실패한 테스트가 있으면 원인을 분석해서 quiz_game.py를 수정해줘.
```
